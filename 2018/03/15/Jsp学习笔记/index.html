第二章 jsp基本语法
一、jsp简介

  1. 根本是一个简化的servlet设计，实现了在java中使用html标签。Jsp是一种动态网页技术标准也是Javaee的标准。Jsp与servlet一样，是在服务器端执行的。

二、 常用的动态网站开发技术

  1. jsp：java平台，安全性高，适合大型企业级的web应用程序。
  2. asp：简单易学，但是安全性和跨平台性差。
  3. php：简单，高效，成本低开发周期短，特别适合中小型企业（linux+apache+mysql+php）

三、jsp页面元素构成

  1. jsp页面元素构成：
    静态内容 指令 表达式 小脚本 声明 注释
  2. jsp指令：
    A． page指令：通常位于jsp页面的顶端，同一个页面可以有多个page指令。
    <%@ page 属性1=”属性值1” 属性2=”属性值2”…. 属性n=”属性值n”%>
    B． include指令：将一个外部文件嵌入到当前jsp文件中，同时解析这个页面中的jsp语句
    C． taglib指令：使用标签库定义新的自定义标签，在jsp页面中启动定制行为。
  3. jsp注释
    A． html注释：<!—html–> //客户端可见
    B． jsp注释：<%–html注释–%>//客户端不可见
    C． jsp脚本注释：//单行注释 /**/ 多行注释
  4. jsp脚本
    在jsp页面中执行的java代码：<%java代码%>
  5. jsp声明
    在jsp中定义变量或者方法：<%！java代码%>
  6. jsp表达式
    在jsp页面中执行的表达式：<%=表达式%>//注意表达式不以分号结束

第二章jsp内置对象

一.内置对象简介

  1. jsp内置对象是web容器创建的一组对象,不用new就可以使用的对象
  2. 常用的jsp内置对象:out request response session application page pagecontest exception config
  3. session对象:在第一个jsp页面被装载时自动创建,完成会话期管理.从一个客户打开浏览器并连接到服务器开始.到客户关闭浏览器离开这个服务器结束,被称为一个会话.当一个客户访问一个服务器时,可能会在服务器的几个页面之间切换,服务器应当通过某种办法知道这是一个客户,就需要session对象
  4. 什么是缓冲区:Buffer,所谓缓冲区就是内存的一块区域用来保存临时数据.
  5. out对象:out对象是jspWriter类的实例,是客户端输出内容常用的对象.
  6. get与post区别
    A.表单:

    B.get以明文的方式通过URL提交数据,数据在URL中可以看到.提交的数据最多不超过2kb,安全性较低但效率比post方式高.适合提交数据量不大,安全性也不高的数据,比如:搜索,查询等功能.
    C.post将用户提交的信息封装在html header内.适合提交数据量大,安全性高的用户信息.比如:注册,修改,上传等功能.
  7. request对象:客户端的请求信息被封装在request对象中,通过它才能了解到客户的需求,然后做出相应,它是HTTPServletRequest类的实例,request对象具有请求域,即完成客户端的请求之前,该对象一直有效.
  8. response对象
    是HTTPServletResponse类的实例。Response对象具有页面作用域，即访问一个页面时，该页面内的response对象只能对这次访问有效，其他页面的response对象对当前页面无效。
